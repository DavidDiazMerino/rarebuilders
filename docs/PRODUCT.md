# RareBuilders Product Brief

## One-liner

RareBuilders is a personal opportunity scout for builders who want to find obscure, high-fit hackathons, grants, bounties and challenges before they become crowded.

## Positioning

RareBuilders is not a hackathon directory. Directories optimize for coverage. RareBuilders optimizes for **personal edge**.

It asks:

- Is this opportunity actually for me?
- Is it hidden enough that I have a better chance?
- Can I reuse something from my existing universe?
- Is the reward money, visibility, access, learning, or a useful artifact?
- What should I build if I decide to enter?

## Merino Labs universe

RareBuilders should feel like part of the Merino Labs universe: opinionated software, curiosity across domains, and willingness to follow weird but meaningful leads.

It should be able to say:

- “This is not your normal AI-agent lane, but a CRISPR education challenge is a good wildcard because you can build a safety/documentation tool without pretending to run a lab.”
- “This watchface contest is tiny; it is physically weird but demoable and could become a Merino Labs artifact.”
- “This OpenAI hackathon is overcrowded, but strategically valuable because launching RareBuilders there is self-referential and useful.”

## Core entities

### BuilderProfile

- domains of interest;
- domains explicitly open as wildcards;
- hard no-go zones;
- preferred reward type;
- weekly hours;
- language/region comfort;
- solo/team mode;
- existing projects and reusable assets;
- feedback history: interested, rejected, entered, won, ignored.

### Opportunity

- source;
- source type;

- region/language;
- deadline;
- reward;
- expected participants or discoverability proxy;

- domains;
- effort estimate;
- hiddenness;

### Score

- fit score;
- win signal;
- hiddenness;
- strategic value;
- effort fit;

- explanation.

## MVP connectors

1. Manual URL/drop connector.
2. Devpost connector.
3. GitHub issue/discussion search connector.
4. RSS/newsletter connector.
5. X/Reddit search as assisted/manual import if APIs are not ready.

## Later connectors

- Discord communities where user has access;
- Telegram channels;
- WeChat/manual relay;
- public PDFs from universities/governments;
- Eventbrite/Meetup;
- Kaggle/AIcrowd/DoraHacks/Lablab/MLH;
- web monitor for selected organizations.

## Demo-first path

For the OpenAI Build Week demo, do not depend on brittle live crawling. Use fixture opportunities plus one real import path. The judged product should show:

1. profile setup;
2. opportunity feed;
3. hidden vs crowded comparison;
4. personalized strategy generation;
5. feedback loop changing recommendations.

## Differentiator

The product's unique metric is **Hiddenness**: how non-obvious an opportunity is. It combines source obscurity, participant visibility, language/region saturation, platform, and whether it appears in mainstream hackathon directories.

## North star

Every morning, David gets five opportunities:

- 2 practical;
- 2 weird but plausible;
- 1 wildcard from another world.

Each arrives with a verdict: enter, monitor, ignore, or recycle idea.
